## Introducción
- Def. Elemento de control ubicado entre red local e Internet
- Usos
	- Consolida mecanismos de seguridad
	- Monitoriza eventos y alarmas
	- Implementa VPNs basadas en IPsec
- Limitaciones
	- Ataques que eluden cortafuegos (módem conectados a ISP)
	- Amenazas internas (insiders)
	- Accesos via WLAN
	- Malware importado de red externa
	- Transferencias de archivos importados

## Tipos por Funcionamiento
- **Filtro de paquetes**
	- Conjunto de reglas que permiten o bloquean paquetes entrantes o salientes
	- Basadas en IPs, puertos e interfaces de red
	- Por defecto descartan / permiten todos los no expresamente permitidos / descartados.
	- Ventajas: Rápidos, Transparentes y escalables
	- Debilidades:
		- Vulnerabilidades aplicaciones
		- Funcionalidades registros limitadas
		- No user auth avanzada
		- Crean brechas por mala configuración
	- Soportan ataques de suplantación IP, encaminamiento y fragmentación de cabeceras.
- **Filtro de paquetes con estado**
	- Sin estado permiten todos los puertos elevados (no conocidos)
	- Con estado guardan un directorio de conexiones establecidas
- **Pasarela (gateway) a nivel de aplicación**
	- Actuan como retransmisores (relays) a nivel de aplicación
	- Para usuarios internos auth, establecen conexiones a redes externas en su nombre.
	- Cada aplicación o servicio tiene su código intermediario (proxy)
	- Son más seguras que filtros de paquetes pero tienen sobrecarga.
- **Pasarela (gateway) a nivel de circuito**
	- Si la conexión es permitida establece dos conexiones simultáneas (interna/externa)
	- Funciona a nivel de transporte. No examina el contenido de los paquetes.
---
## Tipos por Ubicación
- **Hosts Bastión**
	- Dispositivo dedicado (linux), switch o router que actúa como pasarela.
	- Es un elemento crítico que debe ser confiable.
	- Versión segura del SO, servicios esenciales, usualmente con auth y uso limitado disco.
	- Suele tener varias interfaces de red para segmentar las redes.
- **Cortafuegos basados en host**
	- Módulo de SW para proteger a host individual
	- Disponibles en muchos SO y suministrados como paquetes adicionales.
	- Comúnmente en servidores.
	- Reglas de filtrado adaptables al host / indepe a topología red / capa protección extra
- **Cortafuegos personales**
	- Controla tráfico entre un PC y la red corporativa/internet
	- Disponibles en muchos SO y suministrados como paquetes adicionales.
	- Menos complejos.
	- Impiden acceso remoto no autorizado y monitorizan salida para bloquear gusanos
