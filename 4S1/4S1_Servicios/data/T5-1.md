## Tipos de Streaming

| Concepto      | Streaming                                                | Tiempo Real / Interactivo                           |
| ------------- | -------------------------------------------------------- | --------------------------------------------------- |
| Datos         | Contenido pre-grabado (VOD)<br>Directo (Live) con buffer | Comunicación bidireccional<br>(llamadas, reuniones) |
| Tipo de comm  | One-to-Many                                              | One-to-One/Few                                      |
| Latencia      | Latencia alta (segundos)                                 | Latencia muy baja (<200ms)                          |
| P. Transporte | TCP                                                      | UDP                                                 |
| Prioridad     | Calidad de imagen/audio                                  | Fluidez / Interactividad                            |
| Ejemplos      | Netfilx, Youtube, Twitch                                 | Zoom, Google Meet, MS Teams                         |

## Contenedores y Codecs
- **Contenedores (Formato/extensión)**
	- Especifican cómo se guardan los datos (imagen, audio, video y metadatos).
	- Pueden contener varios streams (video/audio) con distintos codecs.
	- Ejemplos: `.mp4`, `.mkv`, `.webm`
- **Codecs**
	- Algoritmo matemático de compresión de un único stream.
	- Ejemplos: `AAC`, `OPUS`

## Tipos de Información
### Imágenes 
- **Mapas de bits** (Bitmap o raster)
	- Array de puntos (pixeles)
	- Color representado como n bits (RGB)
	- Se pueden comprimir (redundancia espacial)
		- Sin pérdida (`.bmp`, `.png`, `webp`)
		- Con pérdida (`.jpeg`)
- **Vectorial**
	- Lista de objetos (circulo, recta, arco)
	- Tienen atributos (tamaño, origen, color, grosor)
	- `svg` y otros.
### Audio Digital
- **Representación**
	- Sample Rate (eje temporal)
		- Establece el número de mediciones por segundo
		- Teorema de Nyquist
			- $f_s > 2\times f_{max}$
			- Muestrear al doble de frecuencia máxima para poder reconstruir señal.
		- Estándar: 48 kHz
	- Bit Depth (eje de amplitud)
		- Establece la precisión del sonido (16/24 bit)
		- Estándar: Dinámico (16/24 bit)
- **Codecs**
	- Sin comprimir o *raw* (`PCM` )
	- Compresión sin pérdida o *lossless* (`FLAC`)
	- Codec Modernos (con pérdida)
		- AAC.
			- Estándar para vídeo (`MP4`o `HSL`)
			- Alta eficiencia
			- Propietario (patentes)
		- Opus
			- Estándar para tiempo real (`WebRTC`)
			- Latencia ultra baja
			- Open source
### Video Digital
- Jerarquía de compresión
	- Redundancia Espacial (Inta-frame). Comprime frame independiente (similar a foto)
	- Redundancia Temporal (Inter-frame). Guarda lo que se mueve (GOP - Group of Pictures)
		- `I-frame` (*Keyframe*): Imagen completa, independiente y de mayor tamaño.
		- `P-frame` (*Predicted*): Guarda cambios respecto al anterior.
		- `I-frame` (*Bi-predictive*): Generada a partir de la anterior y posterior.
